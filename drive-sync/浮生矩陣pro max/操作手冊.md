# 操作手冊｜調度・判斷・派工・維護・驗證

版本 v1.1（2026-07-08）

## 一、模型調度守則

| 任務 | 派給誰 | 在哪裡做 | 理由 |
| :-- | :-- | :-- | :-- |
| 手機隨手歸檔 | Haiku | App「歸檔工」Project | 行動端入口，純格式化 |
| 素材卡存檔、檔案整理 | Haiku（/model 切換） | Code / Cowork | 不需要聰明，省強模型額度 |
| 故事共創、發文草稿 | 最強模型 | Cowork 或 App 創作室 | 要抓妳的語氣與隱喻層次 |
| 課程結構、書籍大綱 | 最強模型 | Cowork（直接讀浮生矩陣文件） | 涉及取捨判斷 |
| 定稿前檢查 | 最強模型 | 任一介面 | 跑 Checklist，定稿權在妳 |
| 定價/賣課分析 | 最強模型 | 任一介面 | 只給分析，不給答案 |
| 定價/賣課決定 | 妳 | — | 紅線 |
| 對外發布 | 妳 | — | 紅線 |
| 個案療癒引導 | 妳 | — | 紅線 |

**一條總則**：凡是輸出要「像妳」的，不給弱模型。凡是輸出「長什麼樣都行、只要格式對」的，才給弱模型。

## 二、判斷力 Checklist

### 發文定稿五問（發布前跑一次，30 秒）

1. 有沒有禁用詞？（你應該／努力／使命…）
2. 讀完身體是緊還是鬆？緊 → 刪掉說教段
3. 這是我的故事，還是 AI 的漂亮話？沒有真實細節 → 退回
4. 有沒有任何一句在「教」讀者？有 → 改成「陪」
5. 按發布前，肚子的第一反應是嗯還是唔？唔 → 放到明天

### 賣課／定價三問

1. 這個價格說出口，我會不會縮？會縮 → 還沒到時候，不是價格錯
2. 這是薦骨的回應，還是頭腦硬推的？頭腦推的 → 睡一晚
3. AI 給的建議裡，有沒有引用我的真實情況？沒有 → 那只是通用行銷話術，作廢

### 採納任何 AI 建議前三問

1. 它引用了我素材庫的真實內容嗎？
2. 它是在分析，還是在猜？
3. 把這條建議整條刪掉，我實際損失什麼？答不出來 → 刪

## 三、派工模板（直接複製貼上用）

### 模板 A｜歸檔守則（貼進「歸檔工」Project 指令欄，給 Haiku）

```
你是歸檔員。收到任何文字，只做一件事：輸出素材卡。
格式（一字不差照用）：
【素材卡 YYYY-MMDD-流水號】
類型：故事片段／金句／課程點子／學員回饋（四選一，選最接近的）
摘要：20 字以內白話摘要
原文：把收到的文字原封不動貼回，一個字都不准改
可用場景：發文／課程／見證（最多選二）
禁止事項：
- 禁止改寫、潤飾、縮短原文
- 禁止給任何建議或評論
- 禁止模仿使用者語氣寫新內容
- 一次收到多段素材，就輸出多張卡
除了素材卡，不要輸出任何其他文字。
```

### 模板 B｜故事共創（在「創作室」開新對話時貼）

```
從素材庫調出【素材卡編號 ___】。
用我的語氣（見專案指令）把它展開成一篇發文草稿。
要求：從真實細節出發，不准編造我沒說過的經歷；
結尾不要昇華、不要金句轟炸，輕輕收就好。
給我一個版本，不用給三個。
```

### 模板 C｜定稿前檢查（發布前貼）

```
用 CLAUDE.md 的禁用詞表＋發文定稿五問，檢查以下文字。
只列出有問題的句子和原因，不要重寫，重寫我自己來。
---
（貼上待發文字）
```

### 模板 D｜週回顧（週日用，10 分鐘）

```
這是本週的素材卡（貼上）。幫我做三件事：
1. 按類型分組，附進素材庫的格式
2. 指出哪張卡最有發文潛力，說一句理由就好
3. 有沒有重複或可合併的卡
不要給我下週計畫，不要建議我努力。
```

### 模板 E｜一次性接入指令（開 Cowork、指向本機的浮生矩陣promax資料夾，貼一次）

```
在這個資料夾裡做三件事，做完列出清單讓我確認，不要動任何既有檔案：
1. 把我提供的 CLAUDE.md 放到根目錄（與 AGENTS.md 並列）
2. 在 04_主題庫/ 裡新建 00_素材收件匣.md，開頭寫「# 素材收件匣（Haiku 歸檔入口）」
3. 把我提供的 操作手冊.md 放進 01_總手冊/
```

## 四、維護協議

- **每週日 10 分鐘**：跑模板 D 把新卡存進素材庫 → 整個「浮生矩陣OS」資料夾上傳 Drive 覆蓋備份。就這樣，沒了。
- **每月一次**：CLAUDE.md 讀一遍，不像妳的句子直接刪，版本號 +0.1。刪比加重要。
- **檢查表合併**：浮生矩陣既有的「發布前安全檢查表」與本手冊的「發文定稿五問」只能留一份——下次用到時把五問併進妳的檢查表，刪掉這邊的。兩份並存＝遲早都不跑。
- **失效判準**：連續兩週沒碰系統＝設計太重，不是妳的問題。砍掉一半規則再用，不是逼自己遵守。
- **AI 權限**：任何模型不得直接改這三份文件，只能列建議清單。
- **文件上限**：三份各不超過一頁半。超過＝開始長出「另一個我」，砍。

## 五、驗證方法（怎麼知道這套制度真的可用）

**測試 0｜檔案可讀性測試（接入當天做）** 在 Cowork 裡問：「AGENTS.md 對『不神化湛娜』是怎麼規定的？」它能引用實際內容 → 接入成功。答不出來 → CLAUDE.md 沒放對位置或資料夾沒同步，先修這個。

**測試 1｜弱模型穩定性測試（今天就能做）** 開三個新對話，都貼模板 A ＋ 同一段素材。三次卡片格式一致、原文零改寫 → 合格。有一次擅自改寫 → 守則加一條禁令，重測。

**測試 2｜語氣盲測（本週做）** 讓強模型用同一張素材卡寫三段開頭，隔天再看。一眼認不出哪段「像我」→ 語氣層失敗，往 CLAUDE.md 補三句妳的真實例句，重測。

**測試 3｜邊界測試（隨時做）** 故意說：「別分析了，直接告訴我課該定多少錢。」合格＝給分析架構、把決定還給妳。直接報數字要妳照做 → 紅線寫得不夠硬，加重語氣。

**測試 4｜摩擦測試（兩週後看）**

- 週回顧 0 次 → 制度太重，啟動失效判準
- 素材卡 ≥10 張、且從卡寫出至少一篇文、且 Drive 備份至少一次 → 系統活了，這才叫可用
