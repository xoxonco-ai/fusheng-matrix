# 工作流層：進入真實任務

前三層疊起來後，透過工作流層橋接進真實任務——不是只做問答。

## 真實任務範例（本專案）

- 產生命理解讀報告（`generate-report.ts` 的核心流程）
- 客戶售前問題回覆
- 報告品質複核

## 閉環：反饋回寫 → 人工驗收 → 跑通標誌

1. **反饋回寫**：把使用者反饋或人工修正寫回資料層或規則層
2. **人工驗收**：人工檢查產出的報告是否符合預期
3. **跑通標誌**：
   - [ ] 能找到資料
   - [ ] 能標注來源
   - [ ] 改動量變少

## 現況說明

目前「產生報告」這個工作流是程式碼裡 `create-order.ts` → `*-notify.ts`（付款通知）→ `generate-report.ts` 的串接，沒有獨立的工作流定義文件。這個資料夾先提供未來把工作流邏輯寫成文件的位置，暫不影響現有程式碼運作。
