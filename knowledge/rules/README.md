# 規則層：規則不是一句 prompt

規則要拆成以下維度分別定義，而不是塞進一句 prompt：

- **身份背景**：AI 扮演誰（例如「浮生矩陣的命理解讀作者」）
- **目標讀者**：付費使用者，希望得到的報告調性
- **輸出格式**：報告章節結構（例如現有的「軍團編制」章節設計）
- **語氣風格**：溫和／犀利／狠——目前是 `generate-report.ts` 裡的 tone map
- **引用來源**：命理判斷依據哪個體系/理論
- **禁用表達**：不能用的斷言（例如過度保證性的用詞）、不能做的醫療/財務等專業建議
- **不知道就說不知道**：命盤資訊不足時，AI 必須明確說明限制，不能編造

## 現況與遷移建議

目前這些規則以字串常數的形式寫在 `generate-report.ts`（約 30–140 行）裡：身份設定、語氣 map、報告章節結構都在程式碼內。如果未來要把規則外部化到這個資料夾（例如 `tone-map.md`、`persona.md`、`report-structure.md`），需要同時修改 `generate-report.ts` 讓它讀取這裡的檔案，這是一次獨立的重構工作，不在本次文件骨架的範圍內。
