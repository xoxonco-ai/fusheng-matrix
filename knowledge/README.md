# 知識庫四層架構

這個資料夾定義一套讓 AI（命理解讀報告產生器）能穩定使用資料與規則的四層架構。四層由下往上疊加：

1. [data/](data/README.md) 資料層 — AI 能讀什麼
2. [index/](index/README.md) 索引層 — AI 怎麼找到資料
3. [rules/](rules/README.md) 規則層 — AI 要照什麼規矩輸出
4. [workflow/](workflow/README.md) 工作流層 — 把前三層接進真實任務

## 目前狀態說明

這個資料夾裡的內容是 `generate-report.ts` 現有邏輯（身份設定、語氣 map、報告章節結構、誠實邊界）的**可讀化摘錄**，不是全新設計出來的規則。`generate-report.ts` 本身完全沒被修改，程式仍照舊直接使用程式碼裡的字串常數——這份文件跟程式碼目前是兩份獨立的東西，改一邊不會自動同步到另一邊。

## 使用順序

先把資料層清乾淨，再建索引，再定規則，最後才接工作流。浮生矩陣的資料層本身很單薄（見 [data/README.md](data/README.md)），所以索引層目前也用不上。

## 驗收標準（跑通標誌）

- [ ] 能找到資料
- [ ] 能標注來源
- [ ] 變更規則或資料時，無需修改/重新部署核心程式碼 —— **目前不成立**，詳見 [workflow/README.md](workflow/README.md)
