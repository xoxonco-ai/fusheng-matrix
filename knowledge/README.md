# 知識庫四層架構

這個資料夾定義一套讓 AI（命理解讀報告產生器）能穩定使用資料與規則的四層架構。四層由下往上疊加：

1. `data/` 資料層 — AI 能讀什麼
2. `index/` 索引層 — AI 怎麼找到資料
3. `rules/` 規則層 — AI 要照什麼規矩輸出
4. `workflow/` 工作流層 — 把前三層接進真實任務

## 目前狀態說明

這個資料夾目前只是**規劃/文件用途的骨架**，尚未接入 `generate-report.ts` 的實際產出流程。`generate-report.ts` 現在把 AI 的身份設定、語氣（溫和/犀利/狠）、報告章節結構（「軍團編制」）都寫成程式碼裡的字串常數；`paipan.html`／`hepan.html` 也把命理計算用的參考表格內嵌在頁面裡。這個資料夾提供一個未來可以把這些內容外部化、結構化的目標架構，但目前不會、也不應該自動修改那些既有檔案——外部化需要另外規劃並測試，屬於後續工作。

## 使用順序

先把資料層清乾淨，再建索引，再定規則，最後才接工作流。

## 驗收標準（跑通標誌）

- [ ] 能找到資料
- [ ] 能標注來源
- [ ] 改動量變少
