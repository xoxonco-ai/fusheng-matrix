# 資料層：AI 能讀什麼

## 原則：先乾淨，再變多

AI 只讀「乾淨資料」，不讀「髒亂資料」。

### 髒亂資料（不要放進來）

- 圖片 PDF、排版混亂的掃描件
- 過期或矛盾的版本
- 來源不清的內容

### 乾淨資料（放進這裡）

資料要同時滿足：**乾淨、可讀、可追溯**。

建議分類（針對本專案）：

- `astrology-tables/` 命理計算參考表（紫微斗數／八字／西洋占星／人類圖對照）——目前內嵌在 `paipan.html`／`hepan.html`，是未來可以外部化到這裡的候選內容
- `case-studies/` 過往報告案例（去識別化後）
- `faq/` 常見客戶問題

### 格式建議

- 一律使用 Markdown
- 只放真實資料，不放測試假資料

### 數量建議

初期 10–30 份乾淨資料即可。
