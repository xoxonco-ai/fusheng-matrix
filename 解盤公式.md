# 浮生矩陣｜解盤公式技術文件

本文件公開「人生羅盤」排盤引擎的計算方式。所有命盤都由開源引擎在**你的瀏覽器內即時實算**，不是查表、不是套模板。程式碼位置：`paipan.html`（單人）、`hepan.html`（合盤 `calcPerson()`）、`admin.html`（後台）。

## 使用的開源引擎

| 引擎 | 版本 | 用途 |
|---|---|---|
| [lunar-javascript](https://github.com/6tail/lunar-javascript) | 1.7.7 | 八字：干支、十神、藏干、納音、大運、胎元、命宮、旬空 |
| [iztro](https://github.com/SylarLong/iztro) | 2.5.8 | 紫微斗數：十二宮、主輔星、亮度、四化、大限 |
| [astronomy-engine](https://github.com/cosinekitty/astronomy) | 2.1.19 | 天文位置：行星黃經、恆星時（占星與人類圖共用） |
| [opencc-js](https://github.com/nk2028/opencc-js) | — | 簡繁轉換（iztro 輸出為簡體） |

## 一、八字命理

輸入的國曆生日先轉為 `Solar` 再取 `Lunar` 與 `EightChar`：

```js
const solar = Solar.fromYmdHms(Y, M, D, h, mi, 0);
const lunar = solar.getLunar();
const ec = lunar.getEightChar();   // 四柱
```

- **四柱**：`ec.getYear() / getMonth() / getDay() / getTime()`（勾選「不確定出生時間」時以中午 12:00 推算，時柱標示「待定」）。
- **十神**：`getYearShiShenGan()` 等由引擎依日主推得；日柱天干即日主。
- **藏干／納音**：`getXxxHideGan()`、`getXxxNaYin()`。
- **五行分布**：四天干依 `甲乙=木、丙丁=火、戊己=土、庚辛=金、壬癸=水` 計數；四地支依 `寅卯=木、巳午=火、辰戌丑未=土、申酉=金、亥子=水` 計數，合計 8 個字（時辰不明時 6 個）。
- **大運**：`ec.getYun(gender==='男'?1:0).getDaYun()`，取第 1～8 步，每步含干支與起始年齡。
- **胎元／命宮／身宮／旬空**：`getTaiYuan() / getMingGong() / getShenGong() / getDayXunKong()`。

## 二、紫微斗數

以 iztro 依國曆排盤，關鍵是**時辰索引**換算：

```js
const ti = (h === 23) ? 12 : Math.floor((h + 1) / 2);   // 0=早子時 … 12=晚子時
const zw = iztro.astro.bySolar(`${Y}-${M}-${D}`, ti, gender==='男'?'male':'female', 'zh-CN');
```

- **命宮**：`zw.palaces.find(p => p.name === '命宫')`，主星含亮度（廟旺得利平不陷）與四化（祿權科忌）。
- **命主／身主**：`zw.soul / zw.body`；**五行局**：`zw.fiveElementsClass`。
- **生年四化**：掃描全部宮位星曜的 `s.mutagen` 收集。
- **十二宮**：固定順序 命宮→兄弟→夫妻→子女→財帛→疾厄→遷移→僕役→官祿→田宅→福德→父母，各宮附天干地支與大限區間 `p.decadal.range`。

## 三、西洋占星

行星黃經由 astronomy-engine 實算（地心視黃經）：

```js
const planetLon = (body, date) => Astronomy.Ecliptic(Astronomy.GeoVector(body, date, true)).elon;
```

**上升（ASC）與天頂（MC）** 用標準球面天文公式，先把出生時間轉 UTC（`h - tz`），再：

```js
const gst  = Astronomy.SiderealTime(utc);              // 格林威治恆星時（小時）
const ramc = ((gst + lon/15) * 15) % 360;              // 天頂赤經 RAMC（度）
const Tc   = (utc/864e5 + 2440587.5 - 2451545) / 36525; // 儒略世紀數
const eps  = (23.439291 - 0.0130042 * Tc) * π/180;      // 黃赤交角
// ASC = atan2( cos(RAMC), -( sin(RAMC)·cos(ε) + tan(φ)·sin(ε) ) )
// MC  = atan2( sin(RAMC),  cos(RAMC)·cos(ε) )
```

φ 為出生地緯度，結果正規化到 0–360°，每 30° 一個星座。

- **宮位**：整宮制（Whole Sign）——上升星座為第一宮，行星所在星座相對上升星座的偏移即宮位：`house = ((sign - ascSign) % 12 + 12) % 12`。
- **逆行**：比較當日與次日黃經，Δ<0 即逆行（日月除外）。
- **相位**：合 0°(容許 8°)、六分 60°(4°)、四分 90°(6°)、三分 120°(6°)、對分 180°(8°)。
- **元素／性質分布**：十行星依星座歸入 火土風水 四元素與 開創/固定/變動 三性質。

## 四、人類圖

### 黃經 → 閘門

人類圖 64 閘門在黃道上的排列以 **302°（水瓶 2°）為 41 號閘門起點**，每閘門佔 5.625°、每爻佔 0.9375°：

```js
const GATE_ORDER = [41,19,13,49,30,55,37,63,22,36,25,17,21,51,42,3,27,24,2,23,8,20,16,35,
                    45,12,15,52,39,53,62,56,31,33,7,4,29,59,40,64,47,6,46,18,48,57,32,50,
                    28,44,1,43,14,34,9,5,26,11,10,58,38,54,61,60];
const lonToGate = l => {
  const d = ((l - 302) % 360 + 360) % 360;
  const i = Math.floor(d / 5.625);
  return { gate: GATE_ORDER[i], line: Math.floor((d - i*5.625) / 0.9375) + 1 };
};
```

### 設計盤（Design）

設計盤取**出生前太陽黃經倒退 88° 的時刻**，用二分法在出生前 80～95 天之間逼近：

```js
const tg = (sunLon(utc) - 88 + 360) % 360;
let lo = new Date(utc - 95*864e5), hi = new Date(utc - 80*864e5);
for (let i = 0; i < 60; i++) {
  const m = new Date((+lo + +hi) / 2);
  const df = ((sunLon(m) - tg + 540) % 360) - 180;
  if (df > 0) hi = m; else lo = m;
}
const design = new Date((+lo + +hi) / 2);
```

### 啟動點

個性盤（出生時刻）與設計盤各取 13 個天體：太陽、地球（=太陽+180°）、北交點、南交點（=北交+180°）、月亮、水星、金星、火星、木星、土星、天王星、海王星、冥王星，各自換算成 `閘門.爻`，共 26 個啟動。

### 通道、中心、類型、權威

- **通道**：36 條固定通道表（1-8、2-14、3-60…），兩端閘門皆被啟動即成立。
- **定義中心**：成立通道兩端閘門所屬中心即被定義；用鄰接表 + BFS 判斷中心之間是否連通。
- **類型判定**（依序）：
  1. 無定義中心 → **反映者**
  2. 薦骨有定義 → 馬達（薦骨/意志/情緒/根部）連到喉嚨 → **顯示生產者**，否則 **生產者**
  3. 薦骨無定義 → 意志/情緒/根部連到喉嚨 → **顯示者**，否則 **投射者**
- **內在權威**（優先序）：情緒中心 → 薦骨 → 直覺（脾）→ 意志力（心）→ 自我投射（G）→（反映者=月循環，其餘=心智/環境）。
- **人生角色**：個性太陽爻 / 設計太陽爻（如 5/1）。
- **定義**：連通分量數 → 一分/二分/三分/四分人。
- **輪迴交叉**：個性日地閘門 · 設計日地閘門。

## 五、關卡年（可驗證佐證）

報告中「你在西元＿＿年前後會經歷…」的年份不是編的，來自三個可實算的訊號（`chartSummary()` 內）：

1. **土星回歸**：實算本命土星黃經，掃描 27～32 歲（步長 0.05 年）找行運土星回到本命位置的年份 → 人生結構性重整期。
2. **流年沖日支**：日支的對沖地支（+6），逐年檢查 `((年-4) % 12)` 對應地支，列出 14～55 歲間所有正沖年 → 變動、低谷或轉折，取前後一兩年。
3. **大運交接**：lunar-javascript 大運起始年（16～56 歲間），每次交接為人生階段大轉換。

AI 撰寫時被要求從中挑 2～3 個最關鍵年份寫進報告，讓讀者可以拿真實經歷驗證。

## 六、父母線索（佐證用）

- **八字**：生我者為印（=母親議題），我剋者為財（=父親議題），統計八字中印星與財星數量——0 個＝緣薄/滋養缺位，多個＝影響強/有張力。
- **紫微**：父母宮主星。
- 規則：只談關係與性格議題，嚴禁猜測姓氏或具體事件。

## 七、合盤（hepan.html）

兩人各自跑一次上述完整實算（`calcPerson()`），再做結構比對產出「合盤速讀」：日主生剋、五行互補/衝突、太陽月亮上升相位（合/三分/六分=同頻，四分/對分=張力）、人類圖電磁通道（一人一半閘門合成完整通道）、類型互動（如生產者×投射者）。完整合盤報告由 AI 依兩張命盤摘要＋關係階段（曖昧探索/熱戀磨合/穩定婚姻/修復期）生成「同頻版＋碰撞版」。

## 八、誠實聲明

- 排盤是**確定性計算**：同樣的出生資料永遠得到同樣的盤，歡迎拿其他排盤軟體對照驗證。
- 出生時間不確定時，以中午推算並明確標示受影響項目（八字時柱、紫微命宮、上升、人類圖類型）僅供參考。
- 命盤描述的是結構與慣性，不是命運宣判；解讀內容僅供自我探索參考。
